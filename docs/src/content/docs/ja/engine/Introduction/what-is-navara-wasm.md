## navara_wasm とは？

navara_wasm は、Rust 製の GIS エンジンを WebAssembly（WASM）化し、Web ブラウザで動作可能にするモジュールです。レンダリングエンジンと非同期に通信しながら、高速な地図処理を行います。

## 主な役割

- Rust 製の GIS エンジンをブラウザで動かすためのエントリーポイント
- レンダリングエンジン（例：navara_three）との非同期通信を管理
- LOD、座標変換、タイル管理などのGIS処理を内部で実行
- 内部の状態を Bevy ECS で管理・更新

## 特徴

- Rust 製エンジン

  - 型安全性による堅牢な実装を実現
  - 低レベルなメモリ管理による最適化

- レンダリングエンジンとの効率的な連携

  - navara_three などからの命令をイベント駆動で受け取り、状態更新や地物の追加を非同期で処理
  - 高負荷処理をWeb workerへ委譲し、メインスレッドの負荷を軽減

- ECS による効率的な状態管理

  - Bevy ECS を用いて、エンティティ（地物、カメラ、地形など）を柔軟に制御
  - GISデータに応じた LOD や可視範囲の自動制御が可能

- レンダリングエンジンに依存しない構成

  - レンダリングエンジンエンジン（Three.js や Babylon.js）に依存せず、汎用性が高い
  - GISエンジンのAPIを独立して使用可能

- マルチプラットフォーム対応が可能
  - 将来的に Web 以外（ネイティブアプリやモバイル）への展開も可能な構成

## 典型的な用途

navara_threeのようなレンダリングエンジン用のライブラリを通して内部的に実行されます。
